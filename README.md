## 프로젝트 소개

BBARU는 단순히 "가장 빠른 길"을 안내하는 서비스가 아니라, 사용자가 **원하는 도착 시각에 가장 가깝게 도착할 수 있도록** 출발 시각과 이동 판단을 도와주는 모바일 UX 기반 웹 프로토타입입니다.

기존 지도 서비스는 현재 시점 기준 경로와 ETA를 보여주는 데 집중하는 반면, BBARU는 다음 질문에 답하는 데 초점을 맞췄습니다.

- 지금 출발해야 하는가?
- 조금 기다렸다가 출발하는 게 더 나은가?
- 현재 속도로 이동하면 정시에 도착할 수 있는가?

이 프로젝트는 공공데이터 활용 관점에서, **교통 신호와 이동 맥락을 사용자 행동 추천으로 바꾸는 경험**을 만드는 데 집중했습니다.

## 문제 정의

이동 서비스에서 사용자가 진짜로 원하는 것은 "최단 시간" 자체보다도 **너무 늦지도, 너무 빠르지도 않게 도착하는 것**에 가깝습니다.

특히 출근, 통학, 면접, 병원 예약, 시험, 공연, 약속처럼 **도착 시각이 중요한 상황**에서는 단순 ETA보다:

- 권장 출발 시각
- 예상 도착 편차
- 현재 행동 지시

가 더 직접적인 가치를 만듭니다.

BBARU는 이 지점을 해결하기 위해 기획된 서비스입니다.

## 핵심 기능

- **목표 도착 시각 기반 출발 추천**
  - 출발지, 도착지, 목표 도착 시각을 입력하면 권장 출발 시각과 예상 도착 시각을 함께 제공합니다.
- **초개인화 ETA**
  - 개인 보폭, 평균 보행 속도, 현재 이동 페이스, 환승 여유 성향 등을 반영해 같은 경로라도 사용자별로 다른 판단을 제공합니다.
- **행동 중심 인터페이스**
  - "지금 출발", "2분 후 출발 권장", "현재 속도 유지", "조금 더 빠르게 이동" 같은 지시를 카드 형태로 보여줘 즉시 행동할 수 있도록 설계했습니다.
- **이동 중 재계산**
  - 횡단보도, 승강장 진입, 환승 지점 등에서 ETA를 다시 계산하고 현재 상황에 맞는 행동을 재추천합니다.
- **도착 전략 비교**
  - `안전 우선`, `균형`, `정시 우선` 모드를 비교해 사용자가 자신의 성향에 맞는 전략을 선택할 수 있도록 구성했습니다.

## 차별점

- **경로 최적화보다 도착 시각 최적화에 집중**
  - 빠른 길 안내를 넘어, 목표 도착 시각에 가장 가깝게 맞추는 것을 핵심 가치로 삼았습니다.
- **데이터 조회형이 아닌 의사결정 지원형 UX**
  - 복잡한 교통 데이터를 사용자가 직접 해석하지 않아도 되도록, 행동 지시를 우선 노출하는 구조를 설계했습니다.
- **출발 전 + 이동 중 전체 흐름을 설계**
  - 한 번의 경로 추천으로 끝나지 않고, 이동 중에도 재계산과 행동 추천이 이어지는 흐름을 구현했습니다.
- **공공데이터를 체감 가능한 사용자 경험으로 전환**
  - 신호 정보와 이동 데이터를 단순 조회가 아닌 실제 판단 보조 정보로 가공했습니다.

```mermaid
    E --> F["이동 중 재계산"]
    F --> G["현재 속도 유지 / 대기 / 속도 조절 추천"]
```

## 구현 범위

현재 결과물은 **React + Vite 기반 모바일 웹 프로토타입**이며, 아래 화면 흐름을 중심으로 구성되어 있습니다.

- 메인 화면
- 경로 결과 화면
- 이동 중 재계산 화면
- 디자인 시스템 가이드 화면

모바일 앱 UX를 기준으로 설계했으며, 시연 가능한 웹 형태로 배포되어 있습니다.

## 기술 스택

- **Frontend**: React, Vite
- **Map / Mobility**: Tmap JavaScript API
- **Deployment**: Vercel
- **UI Direction**: Mobile-first web prototype, action-driven UX

## 공공데이터 활용

이 프로젝트는 공공데이터포털의 **교통안전 신호등 실시간 정보**를 핵심 활용 대상으로 삼았습니다.

- 횡단보도 대기 여부 판단
- 신호 잔여시간 기반 ETA 재계산
- "이번 신호를 건널지 / 다음 신호를 기다릴지"에 대한 행동 판단 보조

여기에 지도 API, 지하철/환승 정보, 실시간 탑승 정보 등을 결합하면 실제 생활 동선 수준에서 더 정교한 ETA 엔진으로 확장할 수 있습니다.

## 포트폴리오 포인트

- **문제 정의가 분명한 서비스 기획**
  - "빠르게 도착"이 아니라 "원하는 시각에 정확히 도착"이라는 사용자 문제에 집중했습니다.
- **데이터를 UX로 번역하는 설계 역량**
  - ETA, 신호, 속도, 이동 맥락을 숫자 나열이 아닌 행동 추천 경험으로 변환했습니다.
- **실서비스 확장을 고려한 구조**
  - 개인화 프로필, 실시간 신호, 환승 버퍼 계산, 혼잡도/날씨 반영 등으로 자연스럽게 확장 가능한 구조를 전제로 설계했습니다.
- **모바일 소비자 서비스 관점의 인터페이스**
  - 복잡한 대시보드 대신, 핵심 정보와 행동 지시를 우선 제시하는 모바일 중심 정보 구조를 구현했습니다.

## 기대 효과

- 대중교통 이용자의 지각 가능성을 줄일 수 있습니다.
- 너무 이른 도착으로 인한 불필요한 시간 낭비를 줄일 수 있습니다.
- 사용자의 실제 보행 특성을 반영해 체감 정확도를 높일 수 있습니다.
- 공공데이터를 생활밀착형 서비스 경험으로 전환하는 사례가 될 수 있습니다.
